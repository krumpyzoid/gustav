Project Structure

electron-vite with Hybrid Services + Ports architecture

Main Process (src/main/)
src/main/ index.ts # App lifecycle, window creation ipc/ handlers.ts # Thin IPC handlers → call services channels.ts # Channel name constants + types services/ worktree.service.ts # Create, remove, clean, list session.service.ts # Tmux session lifecycle state.service.ts # Aggregate state, polling theme.service.ts # Load, watch, broadcast config.service.ts # Parse .wt config files registry.service.ts # Repo registry CRUD ports/ git.port.ts # Interface: worktree ops, branch queries tmux.port.ts # Interface: session/window/pane ops filesystem.port.ts # Interface: read, write, copy, watch shell.port.ts # Interface: exec commands (hooks, install) adapters/ git.adapter.ts # Implements GitPort via git CLI tmux.adapter.ts # Implements TmuxPort via tmux CLI fs.adapter.ts # Implements FileSystemPort via node:fs shell.adapter.ts # Implements ShellPort via child_process domain/ types.ts # SessionEntry, ClaudeStatus, WtConfig, etc.
Preload + Renderer (src/)
src/preload/ index.ts # contextBridge, typed API api.d.ts # Window.api type declarations src/renderer/ index.html # Entry HTML main.tsx # React root + providers App.tsx # Layout: sidebar + terminal hooks/ use-state.ts # Subscribe to state-update IPC use-theme.ts # Subscribe to theme-update IPC use-terminal.ts # xterm.js lifecycle components/ sidebar/ Sidebar.tsx # Session list with repo groups RepoGroup.tsx # Collapsible repo header + entries SessionEntry.tsx # Single session row StatusDot.tsx # Claude status indicator terminal/ Terminal.tsx # xterm.js wrapper component ResizeHandle.tsx # Draggable divider dialogs/ NewWorktreeDialog.tsx # Branch, base ref, options CleanWorktreesDialog.tsx# Cross-repo stale cleanup ConfirmDialog.tsx # Reusable confirm/cancel NewSessionDialog.tsx # Standalone session creation lib/ utils.ts # shadcn cn() utility styles/ globals.css # Tailwind base + theme vars

Root config files

electron.vite.config.ts # electron-vite: main + preload + renderer tsconfig.json # Base TS config tsconfig.main.json # Main process (Node) tsconfig.preload.json # Preload (restricted Node) tsconfig.renderer.json # Renderer (browser + React) tailwind.config.ts # Tailwind with theme vars components.json # shadcn/ui config (Base UI) forge.config.ts # Electron Forge packaging package.json

Key decisions

  • electron-vite replaces bun build — unified HMR for main + renderer
  • Shared types in src/main/domain/types.ts imported by preload for type-safe IPC
  • Services depend on ports (interfaces), never on adapters directly
  • IPC handlers are thin: validate args → call service → return Result type
  • React + Zustand in renderer for state, shadcn/ui (Base UI) for components
