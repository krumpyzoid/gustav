Migration Phases
Phase 1: Scaffold
  • electron-vite project setup
  • React + Tailwind + shadcn init
  • TypeScript configs (3 targets)
  • Shared domain types
  • Port interfaces defined
Phase 2: Main Process
  • Adapters (git, tmux, fs, shell)
  • Services (worktree, session, state, theme, config, registry)
  • Typed IPC handlers
  • Preload bridge with types
  • All wt logic internalized
Phase 3: Renderer
  • React components (sidebar, terminal)
  • shadcn/ui dialogs & selects
  • Zustand state management
  • Theme integration (CSS vars → Tailwind)
  • All confirm() → ConfirmDialog
Phase 4: Package
  • Electron Forge config
  • Remove wt dependency
  • Test on clean machine
  • Build artifacts for Linux

Notes

  • Full rewrite, not incremental migration — the current 3-file structure doesn't lend itself to gradual refactoring. We scaffold the new structure and port logic into it.
  • Registry format unchanged — ~/.local/share/wt/repos.json stays compatible so existing repos are picked up automatically.
  • .wt config is backward compatible — new [clean] section is additive. Repos without it default to origin/staging.
  • No wt CLI needed at runtime — all git worktree, branch, and tmux operations are done directly by Gustav's adapters.
