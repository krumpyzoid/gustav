Design decisions

  • All commands return Result<T> — no more silent catch blocks. Errors surface in the UI (dialog toasts, inline messages).
  • Queries are separate from commands — get-branches feeds the base-ref select in the new worktree dialog, fetched on dialog open.
  • get-clean-candidates is cross-repo — iterates all registered repos, checks each worktree against its repo's .wt merge target.
  • State polling stays at 5s but commands trigger an immediate refresh after completion for snappy feedback.
  • PTY streams remain fire-and-forget (send/on) — no Result type needed for terminal I/O.
